# NDA 검토 실행 워크플로우

## 입력 처리

1. 문서 수신 (파일 업로드 또는 텍스트 붙여넣기)
2. 계약서에서 다음 정보 추출:
   - 계약명 (title)
   - 계약 유형 (단독/상호)
   - 당사자 정보 (A사, B사 등)
   - 목적(Purpose) 조항
   - 비밀정보 정의(Definition) 조항
   - 유효기간/Term 조항
   - 준거법/Governing Law 조항
   - 책임/Liability 조항
   - 데이터 유형 (일반비밀정보/개인정보/의료데이터)

## 검토 프로세스

### 단계 1: 계약서 기본 구조 분석

```
계약서 구조를 파악하고 다음을 식별:
- 전체 조항 수 및 구조
- 주요 조항 위치 (문서 내 라인/항 번호)
- 상대방 정보 (본사 소재지)
- 정보 흐름 방향 (단방향/양방향)
```

### 단계 2: 일반 규칙 적용 (R01-R15)

#### R01: 비밀정보 정의 검토
```
1. "비밀정보" 정의 조항 찾기
2. 다음 표현 체크:
   - "모든 형태" / "all forms" / "일체의 정보" → ❌Too broad
   - 서면 "Confidential" 표시 요건 → ✅권장 또는 ❌누락
   - 구두 공개 시 30일 내 서면 확인 → ✅권장 또는 ❌누락
   - "비밀로 표시되었거나 합리적으로 비밀로 인식될 수 있는 정보" → ✅적정
3. 결과 판정:
   - ❌Too broad → High 수정 필요
   - ❌Missing 표시 요건 → Medium 추가 권장
   - ✅적정 → 통과
```

#### R02: 예외 조항 검토
```
4대 예외 체크:
1. ✅ 공개 정보 (Publicly Available)
2. ✅ 기존 보유 정보 (Prior Possession)
3. ✅ 제3자 적법 취득 (Third-Party Lawful Acquisition)
4. ✅ 법적 의무 (Legal Compulsion)
5. ⚠️ 독자개발 예외 (Independently Developed) → ⚠️ 100% 누락 발견됨

Result:
- 4대 예외 + 독자개발 모두 있음 → ✅ 통과
- 4대 예외만 있고 독자개발 누락 → ❌ High 수정 필요
```

#### R03: 비밀유지 기간 검토
```
1. "비밀유지 기간" 또는 "Confidentiality Period" 찾기
2. 기간 확인:
   - 3년 이하 → ✅적정
   - 3-5년 → ⚠️Medium 협상 권장
   - 5년 초과 → ❌High 단축 협상 필요
   - 무제한 → 🔍영업비밀 별도 규정 확인 필요
3. 영업비밀 별도 규정:
   - Separate provision 있음 → ✅적정
   - 별도 규정 없음 → ⚠️Medium 추가 권장
```

#### R04: 책임상한 검토
```
1. "책임", "Liability", "손해배상" 조항 찾기
2. 다음 표현 체크:
   - "직접·간접·결과적 손해 일체" → ❌무제한 노출
   - "직접손해만 배상" → ✅적정
   - 책임상한 금액 명시 → ✅적정
   - 책임상한 부재 → ❌High 추가 협상 필요
3. 상호 NDA인 경우 상호 Cap 도입 권장
```

#### R05: 인력 비권유 기간 검토
```
1. "Non-solicitation", "인력 비권유", "채용 제한" 조항 찾기
2. 기간 확인:
   - 12개월 이하 → ✅적정
   - 12-24개월 → ❌Medium 단축 협상 필요
   - 24개월 초과 → ❌High 단축 협상 필요 (법적 무효 가능성)
3. 적용 대상 확인:
   - "모든 임직원" → ❌너무 광범위
   - "본 계약으로 알게 된 임직원" → ✅적정
```

#### R06: 준거법 검토
```
1. "Governing Law", "준거법", "관할" 조항 찾기
2. 현재 준거법 확인:
   - 대한민국법 → ✅최우선
   - 싱가포르 중재(SIAC) → ✅중립적
   - 상대방 본사 소재지 법(미국 주법 등) → ❌반드시 협상 필요
3. 관할 법원 확인:
   - 서울중앙지방법원 → ✅최우선
   - 상대방 본사 인근 법원 → ❌협상 필요
```

#### R07: 자문사/임직원 공개 검토
```
1. "자문사", "Advisor", "Employee", "임직원" 관련 조항 찾기
2. 사전 서면 동의 요건:
   - "사전 서면 동의 필요" → ❌실무상 불리
   - "동등 비밀유지 의무 부과 시 동의 없이 가능" → ✅권장
3. 자문사 범위: 회계·법률·기술 전문가 포함 확인
```

#### R08: 목적 조항 검토
```
1. "Purpose", "목적", "거래목적" 조항 찾기
2. 목적 표현 확인:
   - "any business discussion" / "일체의 사업 협의" → ❌너무 광범위
   - "M&A 실사" / "SaaS API 통합 검토" / 구체적 목적 → ✅권장
3. 구체적 목적으로 한정 권장
```

#### R09: 통지 조항 검토
```
1. "Notice", "통지", "공지" 조항 찾기
2. 통지 방법 명시:
   - 서면/이메일 방법 명시 → ✅
   - 미명시 → ⚠️Low 추가 권장
3. 도달 시점 기준:
   - "발신 5영업일 후 도달 의제" → ✅권장
   - "수신 확인 회신" 조건 → ⚠️불명확
```

#### R10: No-Binding 조항 검토
```
1. "No binding obligation", "거래 의무 없음", "협상 의무 없음" 조항 찾기
2. 존재 여부:
   - 있음 → ✅통과
   - 없음 → ❌High 추가 협상 필요
3. 필수 내용 확인:
   - "거래 의무 발생하지 않음"
   - "언제든 협상 중단 가능"
```

#### R11: 상호 NDA vs 정보 비대칭 검토
```
1. "Mutual" / "상호" 표기 확인
2. 정보 흐름 방향 분석:
   - 양측 대등 → ✅Mutual 적정
   - 우리가 많이 공개 → ⚠️단독 NDA 전환 고려
3. 정보 공개량 매핑 권장
```

#### R12: Residuals 조항 검토
```
1. "Residuals", "잔존 지식", "기억" 관련 조항 찾기
2. 상태 확인:
   - Residuals 허용 명시 → ✅권장 (기술 협력 시 특히)
   - Residuals 금지 명시 → ✅대안
   - 미명시 → ⚠️Medium 명확화 권장
```

#### R13: 파기 확인서 검토
```
1. "Return", "반환", "Destruction", "파기" 조항 찾기
2. 파기 확인서 제출 요건:
   - 있음 (임원 서명, 30일 이내) → ✅권장
   - 없음 → ⚠️Medium 추가 권장
```

#### R14: Change of Control 검토
```
1. "Change of Control", "양도", "M&A" 관련 조항 찾기
2. Change of Control 규정:
   - 있는 경우 → ✅통과
   - 없는 경우 → ⚠️Medium 추가 권장 (특히 M&A 실사 NDA)
```

#### R15: 반환/파기 기한 검토
```
1. 반환/파기 기한 확인:
   - "즉시" / "지체 없이" → ⚠️Low 구체화 권장
   - 10-30영업일 → ✅적정
```

### 단계 3: 도메인 규칙 적용 (DR01-DR08) — 해당 시

#### 의료/헬스케어 데이터 포함 시

**DR01**: 환자 의료데이터 → 별도 개인정보 처리 위·수탁 계약 필수
**DR02**: 익명화 기준 (K-anonymity, 차분 프라이시) 명시
**DR03**: 구체적 보안 기준 (ISMS-P, ISO 27701) 명시
**DR04**: 해외 이전 시 PIPA Art. 28-8 준수, 환자 동의 또는 국내 리전 한정
**DR06**: IP 귀속 명확화 — 공동 보유 X, 단독 소유 + license-back
**DR07**: 재위탁 시 책임 승계 조항 필수

#### 모든 민감 데이터 공통

**DR05**: 데이터 유출 통지 — "72시간 이내 서면 통지 + 1차 보고서"
**DR08**: 데이터보호 책임보험 가입 의무

### 단계 4: 결과 종합 및 보고서 생성

```
1. 이슈를 severity별로 분류:
   - 🔴 High: 반드시 수정 협상 필요 (규칙 R01, R02, R04, R05, R06, R10, DR01-04, DR06-07)
   - 🟡 Medium: 협상 권장 (규칙 R03, R07, R08, R11, R12, R13, DR05)
   - 🟢 Low: 추가 권장 (규칙 R09, R14, R15, DR08)

2. 각 이슈에 다음 정보 포함:
   - 문제 조항의 원문 인용
   - 매핑된 규칙 번호
   - 위험도 및 설명
   - 구체적 수정 제안 문구

3. 종합 의견 작성:
   - 전체 계약서 평가
   - 협상 우선순위
   - 가장 중요한 수정항목 TOP 3
```

## 공통 주의사항

- **문서 전체 먼저 읽기**: 검토 시작 전 계약서 구조를 먼저 파악
- **원문 인용**: 각 이슈는 계약서에서 문제가 되는 **원문 구절**을 정확히 인용
- **규칙 매핑**: 각 이슈에 매핑된 규칙 번호를 명시
- **수정안 제공**: 단순 지적이 아닌 **구체적 수정 제안 문구** 제공
- **교차 검증**: 여러 규칙이 중첩되는 경우 모두 명시
- **상호성 고려**: 상호 NDA인 경우 양측 관점 모두 검토
- **법령 참조**: 가능하면 관련 법령 조항 명시 (PIP A, 의료법 등)
